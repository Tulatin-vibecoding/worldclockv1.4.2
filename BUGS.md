# v1.4.2 已知问题

## 1. Android 快速双指缩放跳帧 (未修复)
- **现象**: 快速双指缩放时不跟手，跳过多帧直接显示最终缩放结果
- **原因**: 触控采样 120Hz + 帧率限制 30fps → 3/4 帧丢弃
- **已做**: Pointer Events 替代 Touch Events、交互时临时放开 fpsCap、comet 交互隐藏
- **待做**: 更精细的帧同步策略（或采用原生滚动惯性）
- **影响**: 所有 Android 设备

## 2. Samsung 通知导致渲染破碎 (未验证)
- **现象**: 收到通知时拖动 globe，渲染出现等角度间隔条纹
- **假定原因**: OneUI 通知动画抢占 EGL 上下文，buffer 数据部分丢失
- **已做**: Capacitor resume 时 rebuildSphere 重建球体
- **待验证**: 需在三星平板 OneUI 5.1.1 上实际测试

## 3. 时差文字溢出 (未修复)
- **现象**: 平板纵向时差区域长城市名（如"符拉迪沃斯托克比布宜诺斯艾利斯快xx小时"）不换行，撑大页面
- **已尝试**: word-break、flex min-width:0、overflow-x:hidden、max-width 限制，均无效或引入新问题
- **根因**: flex 布局 + `flex-shrink:0` 自循环，CSS 方法到极限
- **待做**: 可能需要 JS 层面截断城市名或改用不同布局结构

## 4. 全屏 globe 首帧白屏 (未修复)
- **现象**: 全屏按钮第一次打开时 canvas 白屏，需等 1-2 秒才渲染
- **已尝试**: 延迟 resize、forceRender、Fullscreen API 切换
- **待做**: 可能是 WebGL 上下文在全屏过渡时未就绪

## 5. 触控调试功能 (已移除)
- 触摸事件日志因变量作用域问题导致全局触控失灵，调试代码已从事件处理器中移除
- 开关 UI 保留但无功能

## 6. 首次安装页脚文字不刷新 (未修复)
- **现象**: 首次安装打开 App，主页面最下方文字（版权/版本/API 统计）不显示或显示空白，需退出重进才正常
- **可能原因**: WebView 首帧渲染时 DOM 未完全就绪，或 JS 初始化与 CSS 过渡竞态
- **待排查**: 是否为 Service Worker 缓存旧版本 HTML 导致
