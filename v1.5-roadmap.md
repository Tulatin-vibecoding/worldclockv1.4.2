# v1.5 版本更新内容

> 目标：代码归一化 + 架构拆分，不动功能逻辑。
> 基于 v1.4 的技术债清单。

---

## 🔧 重构

### 1. i18n 包装统一
- 当前：`_()` 在 app.js（`const _ = (k) => I.t(k)`）、compare.js（`const _ = (k) => I.t(k)`）、detail.js（`function _(k) { return (W.i18n && W.i18n.t(k)) || k; }`）三种写法
- clock.js / globe.js / mini-globe.js 不用 `_()`，直接内联 `W.i18n ? W.i18n.t(...) : ...`
- 目标：全项目统一为一种 `_()` 模式，放在 `i18n.js` 中挂到 `W._`

### 2. 城市/国家名称格式化独立
- 当前：`formatCityName()` / `formatCountryName()` 嵌在 `i18n.js` 中
- 目标：抽为 `js/name-formatter.js`，独立调用 `W.NameFormatter.city(c)` / `W.NameFormatter.country(c)`

### 3. app.js 拆分（668 行）
- 当前：初始化、搜索、快捷城市、UI 渲染、Dev 面板全部混在一个文件
- 目标：拆为 `init.js` / `search.js` / `ui.js` / `dev-panel.js`

### 4. compare.js 模板分离
- 当前：大型 HTML 模板字符串直接内联在 JS 函数中（buildOverlapResult / buildTimeline / buildPairCards）
- 目标：使用模版标签或独立渲染函数，降低可读成本

### 5. EoT 常量消除
- 当前：`astro.js` 中 `0.0167` 弧度常量出现 5 次，无命名、无注释
- 目标：定义 `const EOT_CONST = 0.0167` 统一引用
